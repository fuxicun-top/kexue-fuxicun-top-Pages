# 项目详细功能介绍

## 一、整体架构

本项目采用前后端分离架构：

- **前端** — 静态页面部署到 GitHub Pages，提供可视化管理界面
- **后端** — 单个 `worker.js` 文件部署到 Cloudflare Workers，处理代理逻辑和 API 请求

## 二、文件功能详解

### 根目录文件

| 文件 | 功能 |
|------|------|
| `worker.js` | **核心后端文件**。包含完整的代理服务逻辑，处理 VLESS/Trojan/Shadowsocks 协议转发、订阅生成、管理后台 API、日志记录、用量统计等所有后端功能。部署到 Cloudflare Workers 后对外提供服务。 |
| `index.html` | 根目录默认页面。当前内容为 nginx 欢迎页占位文件，实际访问由 GitHub Pages 自动指向 `admin/index.html` 或 `login/index.html`。 |
| `README.md` | 项目说明文档，包含项目简介、部署教程、环境变量说明、功能特性等。 |
| `CHANGELOG` | 更新日志文件，记录各版本变更内容和版本号。前端版本检测功能从此文件读取更新历史。 |
| `.gitignore` | Git 忽略规则，排除编辑器配置、系统文件、日志文件等不需要提交的内容。 |
| `locations` | 全球机场位置数据文件（JSON 格式），包含各机场的经纬度、城市名、国家代码、所属大洲等信息。前端地图功能使用此数据展示节点位置。 |
| `sub` | 订阅内容示例文件，包含 Base64 编码的示例订阅链接（如 Trojan 节点配置）。 |
| `version` | 版本信息文件，格式为 `{"Version": 时间戳数字}`。前端管理后台通过此文件检测当前版本，并与 `_worker.js` 中的最新版本对比，显示是否需要更新。 |
| `cdn-cgi/trace` | Cloudflare CDN 追踪信息文件，记录访问参数（IP、TLS 版本、加密套件、数据中心等），用于诊断网络连通性问题。 |

### admin/ 目录（管理后台）

| 文件/目录 | 功能 |
|-----------|------|
| `index.html` | **管理后台主页面**。提供完整的可视化配置界面，包含：用户登录、协议配置（VLESS/Trojan/Shadowsocks）、传输设置（WebSocket/gRPC/TLS）、ProxyIP 优选、SOCKS5/HTTP 链式代理、订阅生成器、Cloudflare 用量统计、Telegram 通知配置、地图节点展示、版本检测与更新日志等功能。 |
| `config.json` | **配置文件**。存储所有代理配置参数，包括 HOST、UUID、传输协议、TLS 设置、订阅配置、反代配置、通知配置、Cloudflare 凭证等。前端页面通过 AJAX 读取和修改此文件。 |
| `log.json` | **访问日志文件**。记录所有用户的登录、配置修改、节点访问等操作日志，包含 IP、ASN、地区、URL、User-Agent、时间戳等信息。 |
| `ADD.txt` | **优选 IP 列表文件**。存储经过测速筛选的优质 IP 地址，包含运营商信息、延迟、带宽等数据，格式为 `IP:端口#地区 运营商 [状态信息]`。 |
| `check` | **网络检测接口**。返回当前网络代理状态、出口 IP、地理位置等信息，用于前端判断网络连接质量。 |
| `getADDAPI` | **添加优选 IP 的 API 接口**。接收前端提交的自定义 IP 数据并写入 `ADD.txt` 文件。 |
| `getCloudflareUsage` | **Cloudflare 用量查询接口**。调用 Cloudflare API 获取 Workers/Pages 的请求量和流量使用情况。 |

### login/ 目录（登录页面）

| 文件 | 功能 |
|------|------|
| `index.html` | **登录页面**。提供密码认证入口，用户输入正确的 ADMIN 密码后才能访问管理后台。包含毛玻璃效果 UI、动画过渡、表单验证等功能。 |

### noADMIN/ 目录（未配置提示页）

| 文件 | 功能 |
|------|------|
| `index.html` | **未设置管理员密码时的提示页面**。当 Worker 未配置 `ADMIN` 环境变量时，用户访问 `/admin` 会被重定向到此页面，提示用户设置环境变量。 |

### noKV/ 目录（未配置 KV 提示页）

| 文件 | 功能 |
|------|------|
| `index.html` | **未绑定 KV 命名空间时的提示页面**。当 Worker 未绑定 KV 存储时，提示用户添加 `KV` 命名空间绑定，否则配置无法持久化保存。 |

### .github/workflows/ 目录（自动化部署）

| 文件 | 功能 |
|------|------|
| `static.yml` | **GitHub Actions 工作流文件**。监听 `main` 分支的 push 事件，自动将静态页面文件（admin、login、noADMIN、noKV 等）打包部署到 GitHub Pages。部署时会排除 `worker.js` 等非前端文件，确保只发布静态资源。 |

## 三、工作流程

### 用户访问流程

1. 用户访问 `https://你的域名/login/` → 输入管理员密码
2. 密码验证通过后 → 跳转到 `https://你的域名/admin/` 管理后台
3. 管理后台加载 `config.json` 显示当前配置
4. 用户修改配置后 → 通过 AJAX 提交到 Worker 后端
5. Worker 将配置保存到 KV 存储并返回更新后的 `config.json`

### 版本检测流程

1. 管理后台从 `version` 文件读取当前版本号
2. 从 `worker.js` 读取最新版本号（`const Version`）
3. 从 `CHANGELOG` 文件读取更新日志
4. 对比两个版本号，显示"最新版本"/"需要更新"/"测试版"状态

### 订阅生成流程

1. 用户在管理后台配置节点参数（协议、传输、TLS 等）
2. 系统根据配置生成 vless:// 格式的节点链接
3. 订阅链接格式：`https://你的域名/你的KEY`，返回 Base64 编码的订阅内容

## 四、关键技术点

- **协议支持**：VLESS、Trojan、Shadowsocks，传输方式支持 WebSocket、gRPC
- **TLS 配置**：支持 ECH/Zero RTT/0-RTT 防断连/TLS 分片（Shadowrocket/Happ）
- **反代优化**：支持 ProxyIP 优选、SOCKS5/HTTP 链式代理、HTTPS/TURN/SSTP 代理
- **数据存储**：Cloudflare KV 用于持久化配置和日志，静态 JSON 文件用于示例数据
- **前端框架**：纯原生 HTML/CSS/JavaScript，无第三方框架依赖
- **CI/CD**：GitHub Actions 自动构建和部署静态页面到 GitHub Pages
